# 启动时清理记录与缓存

这是一个 Chrome Manifest V3 扩展。在 Chrome 启动并加载扩展时，它会根据设置自动清除：

- 浏览历史记录（默认勾选）
- HTTP 缓存文件（默认勾选）
- 全部时间或最近 7、30、90 天的数据

Chrome 的扩展 API 不支持仅删除图片缓存，因此启用“缓存文件”后会清除全部缓存。

## 安装

1. 在 Chrome 地址栏打开 `chrome://extensions/`。
2. 打开右上角的“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择本目录 `chrome-startup-cleaner`。

安装或更新扩展时不会清理数据。只有在完全退出并重新启动 Chrome 时，才会自动执行清理。

## 使用

点击工具栏中的扩展图标，可以：

- 开关历史记录清理
- 开关缓存清理
- 选择全部时间、最近 7 天、最近 30 天或最近 90 天
- 查看上次清理结果
- 点击“打开完整设置页”

也可以打开 `chrome://extensions/`，进入本扩展的“详情”，再点击“扩展程序选项”打开完整设置页。

“最近 7 天”等选项会删除该时间范围内至今产生的数据，不会删除更早的数据。默认清理范围为“全部时间”。

## 验证自动启动清理

1. 访问几个网页。
2. 完全退出所有 Chrome 窗口，确认 Chrome 后台进程也已结束。
3. 重新启动 Chrome。
4. 打开 `chrome://history/`，确认历史记录已被清除。

如果开启了 Chrome 的“关闭 Google Chrome 后继续运行后台应用”，仅关闭窗口可能不会触发浏览器启动事件。
